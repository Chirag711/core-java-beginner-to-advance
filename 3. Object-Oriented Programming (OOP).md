# 3. Object-Oriented Programming (OOP)

Object-Oriented Programming is a programming approach based on real-world entities like objects, classes, and relationships.

---

## 3.1 Classes and Objects (Real-Life Example)

### Real-Life Example: Car

* **Class** → Blueprint (Car design)
* **Object** → Actual car (BMW, Audi)

### Code Example

```java
class Car {
    String brand;
    int speed;

    void start() {
        System.out.println(brand + " is starting");
    }
}

public class Main {
    public static void main(String[] args) {
        Car c1 = new Car();   // object
        c1.brand = "BMW";
        c1.speed = 120;

        c1.start();
    }
}
```

### Output

```
BMW is starting
```

### Explanation

* `Car` is a class (blueprint)
* `c1` is an object (real entity)
* Object uses class properties and methods

## 3.2 Encapsulation (Data Hiding)

### Real-Life Example: Bank Account

* Your bank balance is **hidden**
* You access it using methods (deposit, withdraw)

### Code Example

```java
class BankAccount {
    private int balance = 1000;

    public void deposit(int amount) {
        balance += amount;
    }

    public void showBalance() {
        System.out.println("Balance: " + balance);
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount();
        acc.deposit(500);
        acc.showBalance();
    }
}
```

### Output

```
Balance: 1500
```

### Explanation

* `balance` is private → cannot access directly
* Access controlled via methods
* Ensures security

## 3.3 Inheritance (Reusability)

### Real-Life Example: Vehicle → Car

* Car inherits features from Vehicle

### Code Example

```java
class Vehicle {
    void run() {
        System.out.println("Vehicle is running");
    }
}

class Car extends Vehicle {
    void speed() {
        System.out.println("Car speed is 120 km/h");
    }
}

public class Main {
    public static void main(String[] args) {
        Car c = new Car();
        c.run();
        c.speed();
    }
}
```

### Output

```
Vehicle is running
Car speed is 120 km/h
```

### Explanation

* `Car` inherits from `Vehicle`
* Reuse of existing code

## 3.4 Polymorphism (Many Forms)

### Real-Life Example: Payment System

* Same method → different behavior

  * Credit Card
  * UPI
  * Cash

### Method Overloading Example

```java
class Payment {
    void pay(int amount) {
        System.out.println("Paid using cash: " + amount);
    }

    void pay(String upi) {
        System.out.println("Paid using UPI: " + upi);
    }
}

public class Main {
    public static void main(String[] args) {
        Payment p = new Payment();
        p.pay(500);
        p.pay("chirag@upi");
    }
}
```

### Output

```
Paid using cash: 500
Paid using UPI: chirag@upi
```

---

### Method Overriding Example

```java
class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }

class Dog extends Animal {

### Output

```
Dog barks

## 3.5 Abstraction (Hiding Implementation)

### Real-Life Example: ATM Machine

* You press buttons → get money
* You don’t know internal process

### Code Example

```java
abstract class ATM {
    abstract void withdraw();

    void message() {
        System.out.println("Transaction Processing...");
    }
}

class User extends ATM {
    void withdraw() {
        System.out.println("Cash Withdrawn");
    }
}

public class Main {
    public static void main(String[] args) {
        ATM obj = new User();
        obj.message();
        obj.withdraw();
    }
}
```

### Output

```
Transaction Processing...
Cash Withdrawn
```
